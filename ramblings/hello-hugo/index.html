hello hugo

After a few hours of tinkering with Jekyll for my blog, I've finally made the jump to HUGO. Setup is easy and straightforward, It took me more time in deciding which theme to use and which host to deploy my site than the setup itself.

Installation

Since I'm on Arch Linux, installation is as easy as:

  $ sudo pacman -S hugo

Creating a site

Now that HUGO is installed, create a new site by running

  $ hugo new site clickdot

This command creates all the files necessary to run a fully functional website.

Themes

The next step will be to download a theme or create your own. Totally up to you. You can check out the themes at themes.gohugo.io

Themes can be installed via git or by downloading the archive straight from the author's repo and unpacking it under the themes folder if you don't have git in your system. For git users, you can run:

  $ git submodule add URL_OF_THE_THEME.git themes/themename

This command will download a copy of the theme to your themes folder. Follow your chosen theme's configuration instructions. Some of them only requires you to edit the config file in the root folder of your site.

Creating your first post

To create a draft post simply follow the command

  $ hugo new posts/hello-world.md

You will now have a new file under your contents folder (where all your posts will be saved) named hello-world.md. You can edit this file using a text editor of your choice. I'm using gedit.

All your posts in HUGO will be on the Markdown language. A list of the basic syntax can be found at markdownguide.org/basic-syntax

Running HUGO

It's time to get that site up and running in your local system

  $ hugo server -D

Once it's up and running, head over to http://localhost:1313 on your browser.

Now the fun part begins

Feel free to edit the config, theme and posts as you like.

Creating static

Time to un-draft those posts since drafts don't get deployed. Just replace draft: true

---
title: "hello hugo"
date: 2019-12-28T17:59:52+08:00
draft: true
---

with draft: false

---
title: "hello hugo"
date: 2019-12-28T17:59:52+08:00
draft: false
---

To build the static pages, just run

  $ hugo -D

This will create a folder named public containing all the necessary files for deployment.

Deployment

You are free to choose which host to deploy your site. I recommend Netlify since it's free. Github Pages is also a good choice if you are willing to go through additional steps with the setup. I might do a guide on how to deploy in github soon.
